# Contributing

Thanks for your interest in contributing.

## Ground Rules

- Be respectful and follow `CODE_OF_CONDUCT.md`.
- Keep contributions legal, ethical, and aligned with this repository's compliance-first scope.
- Do not submit code or guidance for covert, unauthorized, or intrusive activities.

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Set up your environment:
   - `python -m venv .venv`
   - `.venv\Scripts\activate`
   - `pip install -r requirements.txt`
4. Run locally:
   - `streamlit run app.py`

## Development Guidelines

- Keep changes focused and small.
- Use clear names and maintain modular structure.
- Update documentation when behavior or setup changes.
- Add or update tests where practical.
- Avoid committing generated artifacts (for example `__pycache__`, local outputs, temp files).

## Commit and PR Expectations

- Write clear commit messages.
- Ensure the app runs before opening a PR.
- Explain:
  - what changed
  - why it changed
  - any limitations or follow-up work
- Link related issues in your PR description.

## Security and Sensitive Reports

For vulnerabilities, follow `SECURITY.md` and do not file public issues with exploit details.

## Licensing

By contributing, you agree that your contributions will be licensed under the repository license in `LICENSE`.
