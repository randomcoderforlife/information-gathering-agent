# Security Policy

## Supported Versions

This project currently supports security updates for:

| Version | Supported |
| ------- | --------- |
| `main`  | Yes       |

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use one of these channels:

1. GitHub Security Advisories (preferred): `Security` tab in this repository, then "Report a vulnerability".
2. If advisory flow is unavailable, contact the repository maintainers directly via repository ownership channels.

## What to Include

- Clear description of the issue and impact
- Steps to reproduce
- Affected files/components
- Suggested remediation (if available)

## Disclosure Process

- We will acknowledge receipt as soon as practical.
- We will investigate and validate the report.
- We will coordinate a fix and release notes where appropriate.
- We will credit reporters when requested and appropriate.

## Scope Notes

This repository is compliance-first and does not support abusive, unauthorized, or intrusive usage patterns. Security reports focused on bypassing legal/safety guardrails may be closed as out of scope.
